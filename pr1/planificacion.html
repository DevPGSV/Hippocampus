Tareas

A la hora de distribuirnos el trabajo, tenemos aproximadamente unas 32 tareas que completar. La gestión, el seguimiento del proyecto y el control de versiones lo realizamos a través de Github, donde también se pueden encontrar las tareas explicadas más detalladamente y quién está asignado a cada una. En la presentación de la memoria final del proyecto, se adjuntará una tabla con la relación del trabajo realizado por cada miembro del grupo y las tareas desarrolladas dentro de la globalidad del proyecto.

Cada miembro del grupo se asigna la tarea con la que se sienta más a gusto o tenga más interés en completar.

Tareas como la estructuración general de la web o el diseño e implementación de la interfaz de usuario las llevan a cabo varias personas, mientras que las tareas de adaptación de servicios ya existentes como Twitter o Github a módulos personalizables son llevadas a cabo por una persona para cada uno de los servicios disponibles (Google Calendar, Campus Virtual, Geaportal, Google Classrom, etc.)

Distribución del trabajo

Para la realización de nuestro proyecto, necesitamos 4 roles de programador principales: front-end, back-end, módulos propios (core) y módulos de terceros (non-core).

El programador front-end se encargará de diseñar e implementar:

  • estructura básica de la web
  • página principal para iniciar sesión o crear una cuenta nueva
  • interfaz de usuario principal (corresponde con la página de inicio con estructura de tablón)

El programador back-end se encargará de implementar:

  • estructura básica de la web
  • asistente de instalación de la aplicación web (similiar al asistente de instalación de Drupal o Wordpress)
  • administración de módulos
  • administración de la página del usuario

Diferenciamos entre dos tipos de módulos que podemos instalar, activar o desactivar en nuestra página personal:

  • módulos propios (core modules)
  • módulos de terceros (non-core modules)

Para más información acerca de estos módulos, se puede consultar la sección de Detalles.

Plazos

Como fechas a destacar, se fija el lunes 3 de abril y el lunes 8 de mayo. Para la primera fecha se espera tener todo el diseño HTML y el estilo CSS completos y correctamente funcionando. Para la segunda fecha se espera tener el diseño completo de las bases de datos y todos los módulos correctamente funcionando. Hay que tener en cuenta de que en caso de existir algún servicio que no cuente con API, nos encargaremos de crear una para poder comunicarnos con el servicio y ofrecer su integración con Hippocampus.

Por último, durante las dos últimas semanas, estaremos realizando pruebas para encontrar fallos y corregirlos de cara a presentar como proyecto final una página completamente funcional.

Además, también será necesaria la redacción de una wiki que recoja las especificaciones detalladas de todas las funcionalidades de la web.
